# UI i18n Polish Audit / Patch Notes - 2026-04-26

## Scope

This document accompanies the approved patch package created after the deep repo audit for **H-Town Combat 67 / Scherbenhimmel**.

The patch intentionally focuses on the remaining player-facing UI string hotspots that were still visible after the first audit pass:

- `app/components/HUD.tsx`
- `app/components/Overlays.tsx`
- `app/components/DebugSuite.tsx`
- `app/components/VersionInfo.tsx`
- `app/components/themed/ThemeDemo.tsx`
- `src/ui/HousingUI.tsx`
- `src/ui/SkillTreeUI.tsx`
- `src/ui/DialogUI.tsx`
- `src/ui/content-browser/ContentBrowser.tsx`
- `src/data/HousingData.ts`
- selected Hub archive/deco/bounty footer literals in `app/components/Hub.tsx`
- package-level strict i18n gate wiring via `scripts/i18n-check-block.mjs`

## Decisions

1. No seamless-open-world concept is introduced.
2. No reward or save logic is moved into UI components.
3. The patch does not create new game systems; it only keyifies visible UI text and tightens the existing single-bounty contract in the Hub.
4. Housing item display strings are converted to stable `nameKey` / `descriptionKey` fields.
5. The strict i18n gate is exposed as `npm run i18n:check:block` and is used from `npm run verify`.
6. The Hub bounty selector is normalized to one selected bounty at a time. Existing saved arrays are sliced to the first selected bounty when the Hub loads.

## Known limitations

This patch is intentionally conservative and does not claim full repository completion.

Still requiring a follow-up implementation pass:

- `src/game/OpenWorld.ts` still exists under active `src/game`. It should be archived or converted into area-instance-only data before the project can be considered architecture-clean.
- `CharacterData`, upgrade data, quest/story data, and broader `src/content/*` files may still contain player-facing data strings. Those require a data-schema migration to `displayNameKey`, `titleKey`, `descriptionKey`, `objectiveKeys`, and related stable keys.
- `SkillTreeUI` localizes the UI shell in this patch, but the actual skill branch/node names still depend on the current character data model.
- `ContentBrowser` localizes the browser shell, filters, tabs, and metadata labels, but it still renders the current content payloads. Full content payload keyification is a separate data-contract task.
- The patch package was prepared without a local Git checkout because the execution environment could not resolve GitHub hosts. Therefore `npm ci`, `npm run verify`, and `npm run build` must be executed in a normal developer checkout before committing.

## Required verification before commit

Run from the repository root after applying the patch:

```bash
npm ci
npm run verify
npm run build
```

If `npm run verify` fails, do not commit. Fix the reported i18n, docs, lint, or test failures first and update `docs/STATUS_VERIFICATION.md` only with commands that were actually executed.
